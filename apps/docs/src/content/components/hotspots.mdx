Overlay clickable regions on any background â€” an image `src`, a component, or an
inline `<svg>`. Hotspot geometry is expressed against a `width` Ă— `height`
coordinate space, so shapes stay aligned as the widget scales. Use the exported
shape primitives (`CircleHotspot`, `RectHotspot`, `EllipseHotspot`,
`PolygonHotspot`) or supply any custom SVG node. Give a `targetId` for a
"find the region" quiz, or omit it for free selection.

<PlaygroundLink
  code={`import { Hotspots, CircleHotspot, EllipseHotspot } from '@freecodecamp/uikit/games';

  const hotspots = [

        hotspots={hotspots}
        targetId='head'
        prompt='Click the head'
        onCorrect={(id) => console.log('correct', id)}
      />
    </div>
  );
}

## Accessibility

Each hotspot is a `role="button"` with `tabindex="0"`, an `aria-label`, and
`aria-pressed` reflecting selection. A visually-hidden `aria-live` region
announces each pick and the result. When `background` is a string, the `<img>`
carries `backgroundAlt`. State colour is paired with a stroke change so it
does not rely on colour alone. The shape transition is suppressed under
`prefers-reduced-motion`.

## Do & Don't

<DoDont>
  <Fragment slot='do'>
    <ul>
      <li>Match `width` / `height` to the background's aspect ratio.</li>
      <li>Give every hotspot a clear, unique `label`.</li>
      <li>Use the shape primitives so state colours apply automatically.</li>
    </ul>
  </Fragment>
  <Fragment slot='dont'>
    <ul>
      <li>Don't overlap hotspots so tightly that picks are ambiguous.</li>
      <li>
        Don't hardcode fill on custom shapes â€” let the component drive state.
      </li>
      <li>
        Don't omit `backgroundAlt` when the background is an informative image.
      </li>
    </ul>
  </Fragment>
</DoDont>
